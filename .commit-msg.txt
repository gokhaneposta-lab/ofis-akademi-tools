fix(home): mobil overflow — KPI kartlari 2x2, formul truncate, grid template virgul

Belirti: ana sayfa mobilde sayfanin sag tarafi kesik / "yarim" goruyordu.

Sebepler:
- Hero icindeki 4'lu KPI kartlari (Toplam Satis, Aylik Artis, Siparis,
  Iade Orani) 'grid-cols-4' ile zorlanmisti. Mobile genisliginde her
  kart icindeki 'text-base' formatlanmis para sayisi (orn. "428.900 TL")
  kart sinirlarini asip tasiyordu.
- Yan tarafta donen Excel formulu (orn. "=COKETOPLA(...;\"Satis\")")
  truncate yoktu, parent flex'i genisletip overflow yapiyordu.
- 'lg:grid-cols-[minmax(0,1.4fr),minmax(0,1fr)]' ve free section'daki
  'sm:grid-cols-[minmax(0,1.5fr),minmax(0,1fr)]' arbitrary value'larinda
  Tailwind'e *track ayraci* olarak virgul gecirilmis. CSS spec'inde
  grid-template-columns track'leri arasinda virgul GECERSIZ; bosluk
  gerekir. Tailwind v4 yazimi: '_' kullanilir.

Duzeltmeler:
- 'grid-cols-4' -> 'grid-cols-2 sm:grid-cols-4' (mobilde 2x2 yerlesim)
- KPI parent ve dashboard kart parent'ina 'min-w-0' eklendi
- Formul span'ina 'min-w-0 truncate' eklendi (asilmaz, kisaltir)
- Iki arbitrary value'da virgul -> '_' donusturuldu, gecerli CSS uretir
